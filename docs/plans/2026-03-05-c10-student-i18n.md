# C10 — Student Component i18n Wiring Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Wire `useTranslations()` into all 24 student-facing components so every hard-coded English string renders from message files, enabling full locale switching.

**Architecture:** Each component imports `useTranslations` from `next-intl`, calls `const t = useTranslations("Namespace")`, and replaces every hard-coded string with `t("key")` or `t("key", { param })` for ICU parameterized strings. 8 pure display components get `"use client"` added. 3 module-level const arrays with labels move inside component bodies. Message files get 101 new keys across 4 new + 3 existing namespaces.

**Tech Stack:** next-intl (useTranslations hook), ICU MessageFormat for parameterized strings, React client components.

**Design doc:** `docs/plans/2026-03-05-c10-student-i18n-design.md`
